# Webiz Inventory ‚Äî Backend API Implementation Plan

## Endpoint

```
POST /api/v1/inventory/checkin
Content-Type: application/json
```

Single endpoint ‚Äî receives one check-in submission, persists it, and returns confirmation.

---

## Payload Fields

### User-submitted (from GUI form)

| Field | Type | Description |
|---|---|---|
| `first_name` | `string` | Employee's first name ‚Äî links the device to a person |
| `last_name` | `string` | Employee's last name ‚Äî used for display and search |
| `email` | `string` | Employee's work email ‚Äî used to send confirmation and to deduplicate person records |
| `project` | `string (enum)` | Which project the employee is assigned to ‚Äî used for project-level device reporting |

### Hardware (collected automatically)

| Field | Type | Description |
|---|---|---|
| `timestamp` | `string (ISO-8601)` | Local time when data was collected on the device ‚Äî used to detect stale/queued submissions |
| `hostname` | `string` | Machine network name ‚Äî used to identify the device on the network |
| `ip_address` | `string` | IP at time of check-in ‚Äî useful for network auditing, not for permanent identification |
| `brand` | `string` | Manufacturer (Apple, Dell, Lenovo‚Ä¶) ‚Äî used for fleet composition reports |
| `model` | `string` | Device model name ‚Äî used for warranty lookups and procurement planning |
| `serial_number` | `string` | Hardware serial ‚Äî **primary device identifier**, used for deduplication and tracking |
| `cpu` | `string` | Processor name and speed ‚Äî used for performance tier classification |
| `ram` | `string` | Total RAM rounded to GB ‚Äî used to flag under-spec devices |
| `storage` | `string` | Disk size (and model on Windows) ‚Äî used for capacity planning |
| `os` | `string` | OS name and version ‚Äî used to track outdated systems and compliance |

### Suggested new fields to add

| Field | Type | Source | Description |
|---|---|---|---|
| `platform` | `string (enum: windows, macos, linux)` | Derived from agent | Enables platform-split reports without parsing the `os` string |
| `agent_version` | `string` | Hardcoded in agent | Tracks which version of the agent submitted ‚Äî useful when rolling out updates |
| `submission_type` | `string (enum: online, queued)` | Agent logic | Whether it was submitted live or flushed from the offline queue ‚Äî helps detect connectivity issues |
| `checkin_id` | `string (UUID)` | Generated by agent before POST | Idempotency key ‚Äî if the agent retries after a network error, the backend can deduplicate |
| `os_version` | `string` | Parsed from `os` field | Separate normalized version string for filtering (e.g. `"14.4.1"` vs `"macOS 14.4.1"`) |

## Database Schema (suggested table: `device_checkins`)

```sql
CREATE TABLE device_checkins (
    id               SERIAL PRIMARY KEY,
    checkin_id       UUID UNIQUE NOT NULL,          -- idempotency key
    received_at      TIMESTAMPTZ DEFAULT NOW(),     -- server-side timestamp
    timestamp        TIMESTAMPTZ NOT NULL,          -- client-reported collection time

    -- Person
    first_name       TEXT NOT NULL,
    last_name        TEXT NOT NULL,
    email            TEXT NOT NULL,
    project          TEXT NOT NULL,

    -- Device identity
    serial_number    TEXT NOT NULL,
    hostname         TEXT NOT NULL,
    brand            TEXT NOT NULL,
    model            TEXT NOT NULL,

    -- Hardware specs
    cpu              TEXT,
    ram              TEXT,
    storage          TEXT,
    ip_address       TEXT,

    -- OS
    os               TEXT NOT NULL,
    os_version       TEXT,
    platform         TEXT,

    -- Agent metadata
    agent_version    TEXT,
    submission_type  TEXT DEFAULT 'online'
);
```

> Index on `serial_number`, `email`, `project`, `platform`, `received_at` ‚Äî these are the filter axes.

## Endpoint Behaviour

```
POST /api/v1/inventory/checkin

‚Üí 200 OK             { "status": "ok", "id": "<checkin_id>" }
‚Üí 409 Conflict       { "status": "duplicate" }            (checkin_id already exists)
‚Üí 422 Unprocessable  { "status": "error", "errors": [...] }
```

**Processing steps:**
1. Validate required fields (`first_name`, `last_name`, `email`, `serial_number`, `timestamp`)
2. Check `checkin_id` uniqueness ‚Äî return `409` if duplicate (handles agent retries)
3. Normalize `os` ‚Üí derive `platform` + `os_version` if not sent by agent
4. Persist to `device_checkins`
5. Optionally upsert a `devices` table keyed on `serial_number` with latest known state
6. Return `{ "status": "ok", "id": "<checkin_id>" }`

## Useful Filters

### By identity

| Filter | Use case |
|---|---|
| `serial_number` | Look up full history of one device |
| `email` | Find all devices owned by one person |
| `hostname` | Quick lookup by machine name |

### By fleet slice

| Filter | Use case |
|---|---|
| `project` | Which devices does project X use |
| `platform` | Windows vs macOS vs Linux split |
| `brand` / `model` | Find all MacBook Pro 14" for warranty batch |
| `os_version` | Identify machines still on an old OS ‚Äî compliance |

### By time

| Filter | Use case |
|---|---|
| `received_at` range | Devices that checked in within a period |
| `received_at` ‚Äî absence | Devices that have NOT checked in for >6 months (overdue) |
| `submission_type = queued` | Devices that were offline during check-in |

### By spec / compliance

| Filter | Use case |
|---|---|
| `ram LIKE "8 GB"` | Flag under-spec devices |
| `os_version < "14"` | Find outdated macOS fleet |
| `platform = windows` + `os LIKE "%10%"` | Windows 10 deprecation audit |

## Agent Updates Required

To support the new fields, add these to the payload in both agents before the POST.

**Python (`inventory_agent.py`):**
```python
import uuid
payload = {
    **user_data, **hw,
    "checkin_id":      str(uuid.uuid4()),
    "agent_version":   "2.0",
    "submission_type": "online",   # set to "queued" when flushing from queue
    "platform":        _sys.lower(),  # "darwin" ‚Üí "macos", "linux", "windows"
}
```

**PowerShell (`WebizInventory_Windows.ps1`):**
```powershell
$payload["checkin_id"]      = [guid]::NewGuid().ToString()
$payload["agent_version"]   = "1.0"
$payload["submission_type"] = "online"
$payload["platform"]        = "windows"
```
